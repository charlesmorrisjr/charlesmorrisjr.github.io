Capstone Project

For my capstone project, I decided to revisit Travlr Getaways, a website I worked on in a previous course that allowed users to book vacation packages. Initially, the website was completely static. I updated the site's Travel section into a full-stack application that pulled information from a MongoDB database. I also added a single-page application allowing site administrators to modify travel packages without coding.

To improve the website further, I decided to enhance its functionality by updating the Rooms section from a static webpage to a full-stack application, similar to what I did for the Travel section. I added a section to the single-page application allowing administrators to modify the available rooms. This project demonstrated my ability to take the knowledge I learned in my studies and implement it in new ways.

Code Review

Before I updated my original project, I performed a code review to find bugs, improve code quality, and ensure that the code meets industry standards. Code review is essential because it helps computer science professionals prevent errors before they happen, identify security vulnerabilities, encourage accountability, and reduce technical debt.

In my previous course, CS 465: Full Stack Development I, I took a static vacation booking website called Travlr Getaways and turned the Travel section into a full-stack application using the MEAN stack. On the customer-facing side, users can view the travel packages that are available to them. These packages are pulled from a database and rendered into a web page for the user to view. A single-page application allows administrators to add, modify, and remove available travel packages. For my capstone project, I decided to change the Room section of the website from a static webpage into a full-stack application similar to the Travel section.

Software Design and Engineering

For this enhancement, I began by working on the customer-facing website. I changed the rooms.html file into a Handlebars template and added JavaScript files in the routes and controllers folders to allow Express to serve the file correctly. The process was relatively straightforward aside from one problem. The header.hbs file always styled the "Travel" menu heading as "selected," even if I was on the "Rooms" page. I had to add code to rooms.js and travel.js in the controllers folder to pass an object named currentPage to header.hbs to designate which menu heading was selected.


The next part of the enhancement involved adding a new page to the single-page application to allow administrators to modify the available rooms to choose from. I had to create the Angular components add-room, edit-room, and rooms. These components are analogous to existing components for displaying, adding, and editing trips. I also added a new link on the navbar so administrators could switch between viewing trips and rooms.

I also updated the styling of the navbar to make it more readable and added cancel buttons to the forms for adding and editing trips and rooms.


Algorithms and Data Structures

For this enhancement, it was necessary to create a new data structure to hold the data for the rooms. This includes each room's code, name, price per day, image link, and description. I created the data structure by taking and modifying the data structure used by the trips in my project since it was similar.

Databases

For this enhancement, I created a new collection in MongoDB to hold data for the Room section of the website. The data included each room's code, name, price per day, image link, and description. Previously, the data was stored in hard-coded JSON files. I added code to retrieve and store the data in the MongoDB database instead, which is displayed on the customer-facing website and the single-page application.
